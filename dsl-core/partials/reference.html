Declaración de Entidades

            
entity <entity name> {
<field name> <type> [<validation>*]
}

  • <entity name> es el nombre de la Entidad,

  • <field name> el nombre de uno de las propiedades de una entidad,

  • <type> los tipos de datos soportados,

  • and as an option <validation> validaciones para las propiedades.

Declaración de Enumeraciones

            
enum <enum name> {
<enum values>
}

  • <enum name> es el nombre de una enum,

  • <enum values> valores separados por coma en Mayúsculas

Declaración de Relaciones

            
relationship <type> {
<from entity>[{<relationship name>}]
to
<to entity>[{<relationship name>}]
}

  • <type> es el tipo de una relación (OneToMany | ManyToOne| OneToOne | ManyToMany)

  • <from entity> es el nombre de una entidad que posee una relación,

  • <to entity> es el nombre de una entidad hacia donde va la relación,

  • <relationship name> es el nombre de la relación en la entidad.

Los posibles tipos y validaciones están descritos aquí aquí, si la validación requiere un valor, simplemente se agrega a la derecha(<value>) del nombre de la validación.

Declaración de Opciones

Es posible declarar algun de las opciones sopotadas a través del DSL, las opciones soportadas son:

  • paginate habilida la paginación en una entidad con pager | pagination | infinite-scroll

  • service habilita la capa de servicio para la entidad con serviceClass | serviceImpl

  • dto habilita la generación de DTO para las entidades

  • skipClient no genera front-end de la entidad

  • skipServer no genera back-end para la entidad

  • angularSuffix agrega un sufijo a los nombres de route en Angular

  • microservice especifica el nombre de la aplicación de una entidad que se encuentra en otro microservicio.

  • search especificar las opciones de busqueda para una entidad que se encuentre en otro servicio

Refer JDL documentation for more details.
